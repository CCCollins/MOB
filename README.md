# Militech Open Bot

Автономный ИИ-агент (Оркестратор), способный взаимодействовать с системой пользователя, искать информацию в интернете и выполнять фоновые задачи через графический интерфейс или в скрытом (Headless) режиме.

## Структура проекта

```
MOB/
├── run.py                  — точка входа (GUI или Headless)
├── config/
│   └── settings.py         — конфигурация (хранится в зашифрованном config.mobcfg)
├── core/
│   ├── agent.py            — ИИ-агент, цикл итераций, инструменты
│   ├── bot_runner.py       — запуск Telegram polling + фоновые задачи
│   ├── database.py         — история чата и долгосрочная память (SQLite)
│   └── tools.py            — инструменты ОС: терминал, мышь, скриншоты, поиск
├── interfaces/
│   ├── gui_app.py          — графический интерфейс (customtkinter)
│   └── telegram_app.py     — Telegram-бот (aiogram)
├── requirements.txt
├── install_linux.sh
├── MOB.spec                — сборка обычной версии (PyInstaller)
└── MOB_Portable.spec       — сборка portable-версии (PyInstaller)
```

## Требования

- Python **3.10+**
- Для GUI: работающий дисплей (X11 на Linux, обычный рабочий стол на Windows/macOS)
- API-ключи: [OpenRouter](https://openrouter.ai/settings/keys) (обязательно), [Brave Search](https://api-dashboard.search.brave.com/app/keys) (для поиска), [Telegram Bot Token](https://t.me/BotFather) (для Telegram)

## Установка и запуск

**Linux**
```bash
chmod +x install_linux.sh
./install_linux.sh
./run.sh
```

**Windows**
```bash
pip install -r requirements.txt
python run.py
```

## Headless-режим (сервер, без графики)

```bash
python run.py --headless
```

В этом режиме бот доступен через Telegram. Инструменты GUI ОС (мышь, скриншоты) автоматически отключаются.

## Настройки

Открываются во вкладке **Настройки** графического интерфейса. Конфигурация хранится в зашифрованном файле `config.mobcfg` — ключ генерируется автоматически на основе параметров машины, пароль не требуется.

| Параметр | Описание |
|---|---|
| Telegram Token | Токен бота от @BotFather |
| Разрешённые TG ID | ID пользователей через запятую |
| OpenRouter API Key | Ключ для доступа к моделям |
| Brave Search API Key | Ключ для веб-поиска |
| DynamicPDF API Key | Конвертация docx/xlsx в PDF (опционально) |
| Proxy | HTTP/SOCKS прокси (опционально) |
| Модели Main / Expert | Основная модель и эксперт для делегирования кода |
| Макс. итераций | Лимит шагов агента за один запрос |
| История | Глубина краткосрочной памяти (сообщений) |
| Фоновая активность | Интервал автономных пробуждений агента |
| Цепочка рассуждений | Накапливать и показывать все шаги вместо перезаписи |

Кнопки в нижней строке:
- **💾 Сохранить** — сохранить текущие настройки
- **📤 Экспорт** — сохранить настройки в зашифрованный `.mobcfg` файл с паролем (для переноса)
- **📥 Импорт** — загрузить настройки из `.mobcfg` файла
- **🔴 Сброс** — мгновенный сброс всех значений к умолчаниям

## Команды чата

| Команда | Описание |
|---|---|
| `/clear` | Очистить краткосрочную память (историю сессии) |
| `/memorize` | Принудительно сохранить факты из диалога в долгосрочную память |
| `/screenshot` | Сделать скриншот экрана |
| `/restart` | Перезапустить бота (только Telegram) |
| `/shutdown` | Выключить процесс бота |

## Portable-версия

Portable-версия хранит все данные (настройки, БД, логи) в папке `data/` рядом с исполняемым файлом — не пишет ничего в `%APPDATA%` или `~/.config`. Удобна для запуска с флешки или без установки.

**Сборка через PyInstaller:**
```bash
# Обычная версия
pyinstaller MOB.spec

# Portable
pyinstaller MOB_Portable.spec
```

Готовая папка `dist/MOB_Portable/` является самодостаточной — достаточно скопировать её целиком. При первом запуске автоматически создаётся `data/`, в которой будут храниться все настройки.

Для сборки инсталлятора portable-версии используется `installer_portable.iss` (Inno Setup) — не требует прав администратора и не регистрируется в системе.

## Сборка обычного инсталлятора (Windows)

```bash
pyinstaller MOB.spec
# затем компилируем installer.iss через Inno Setup
```
